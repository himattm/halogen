# Room Cache Fix & Animated Theme Transitions

## Problem

Two independent issues:

1. **Room cache build failure:** `compileRoomMainKotlinMetadata` fails because the `expect object HalogenDatabaseConstructor` doesn't declare `initialize()` from `RoomDatabaseConstructor<T>`. KSP generates the `actual` implementations correctly for all 4 targets, but the intermediate source set metadata compilation runs first and needs the method signature to satisfy the interface contract.

2. **No theme transition animation:** When `HalogenEngine.activeTheme` changes, the entire Material 3 theme (49 colors, typography, shapes) snaps instantly. This is jarring — colors should smoothly animate to their new values.

## Design

### Part 1: Room Cache Fix

**File:** `halogen-cache-room/src/roomMain/kotlin/halogen/cache/room/db/HalogenDatabase.kt`

Add the `initialize()` method signature to the `expect` declaration:

```kotlin
/**
 * Room KMP database constructor.
 *
 * The `actual` implementations are generated by Room's KSP processor
 * for each target (Android, JVM, iOS). The `@Suppress("NO_ACTUAL_FOR_EXPECT")`
 * is required because the actuals are code-generated, not hand-written.
 * The `initialize()` override must be declared here so that the intermediate
 * `roomMain` source set's metadata compilation can type-check the interface
 * contract before KSP runs.
 */
@Suppress("NO_ACTUAL_FOR_EXPECT")
internal expect object HalogenDatabaseConstructor : RoomDatabaseConstructor<HalogenDatabase> {
    override fun initialize(): HalogenDatabase
}
```

No other changes. KSP continues to generate the actual implementations unchanged.

### Part 2: Animated Theme Transitions

**Module:** `halogen-compose` (only file changed: `HalogenTheme.kt`)

#### API

Two new parameters on `HalogenTheme`:

```kotlin
@Composable
fun HalogenTheme(
    spec: HalogenThemeSpec? = null,
    darkTheme: Boolean = isSystemInDarkTheme(),
    config: HalogenConfig = HalogenConfig.Default,
    colorAnimationSpec: AnimationSpec<Color> = HalogenThemeDefaults.colorAnimation,
    shapeAnimationSpec: AnimationSpec<Float> = HalogenThemeDefaults.shapeAnimation,
    content: @Composable () -> Unit,
)
```

Defaults live in a companion-like object:

```kotlin
object HalogenThemeDefaults {
    val colorAnimation: AnimationSpec<Color> = tween(durationMillis = 400)
    val shapeAnimation: AnimationSpec<Float> = tween(durationMillis = 400)
}
```

Usage:

```kotlin
// Default (smooth 400ms):
HalogenTheme(spec = activeSpec) { MyApp() }

// Custom spring animation:
HalogenTheme(
    spec = activeSpec,
    colorAnimationSpec = spring(stiffness = Spring.StiffnessLow),
    shapeAnimationSpec = spring(stiffness = Spring.StiffnessMedium),
) { MyApp() }

// Disable animation (snap):
HalogenTheme(
    spec = activeSpec,
    colorAnimationSpec = snap(),
) { MyApp() }
```

The second overload (custom `themeWrapper`) receives the same parameters.

#### What Animates

| Property | Animated | Mechanism |
|----------|----------|-----------|
| 49 M3 color roles | Yes | `animateColorAsState(target, colorAnimationSpec)` |
| 5 shape corner radii | Yes | `animateFloatAsState(target, shapeAnimationSpec)` |
| Typography (font, weight, spacing) | No | Instant snap (avoids text reflow during transitions) |

#### Implementation

Inside `HalogenTheme`, after theme expansion completes:

1. **Colors:** Each of the 49 color roles in the target `ColorScheme` is wrapped in `animateColorAsState(targetValue, animationSpec)`. A helper function `ColorScheme.animated(target, spec)` encapsulates this to keep the composable clean.

2. **Shapes:** Each of the 5 corner radii (extraSmall through extraLarge) is wrapped in `animateFloatAsState(targetDp, spec)`, then assembled into `Shapes(RoundedCornerShape(...))`.

3. **First composition:** On first render (no previous theme), `snap()` is used implicitly so the initial theme appears without a 400ms fade-in. This is detected by tracking whether a previous theme has been seen via `remember`.

4. **Dark mode toggle:** Switching dark/light mode also triggers the color animation (the pre-computed light/dark schemes have different colors). This gives a smooth dark mode transition for free.
