# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Older releases | No |

Only the latest published release receives security fixes.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email **security@mmckenna.me** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Affected module(s) and version(s)

### Response Timeline

- **Acknowledgment**: within 48 hours
- **Assessment**: severity evaluation within 1 week
- **Fix**: timeline depends on severity (critical issues are prioritized)

### Credit

Reporters will be credited in the release notes unless they prefer to remain anonymous.
